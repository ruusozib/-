# Brand Lens for Bags

## これは何をするアプリ？（初心者向け）
このアプリは、**ブランドバッグをカメラや画像で読み取って、候補情報を表示する**ためのWebアプリです。

たとえば、スマホでバッグを写して「スキャンする」を押すと、以下のような情報が表示されます。
- どのブランドに近いか（例: CHANEL, DIOR など）
- どのモデルに近いか
- 参考価格帯
- 一致率（どれくらい似ているかの目安）

> 現在は「本番用AIサーバー」ではなく、ブラウザ内の簡易ロジックで推定しています。

---

## 現状の機能（できること）
1. **カメラでスキャン**
   - カメラ開始 / 停止ができる
   - フレーム内の画像を取得して解析できる
2. **画像アップロードでスキャン**
   - 手元の写真を選んで解析できる
3. **結果表示・履歴保存**
   - 上位候補3件、処理時間、ブランド/モデル/価格などを表示
   - スキャン履歴をブラウザに保存・削除できる

---

## 不足している機能（まだ本番に足りない点）
1. **本物の画像認識モデル連携がない**
   - いまは色やコントラスト中心の簡易推定
2. **真贋判定（偽物判定）機能がない**
   - 現在の結果は「似ている候補」であり、鑑定ではない
3. **認証・データ管理・運用監視がない**
   - ユーザー管理、API保護、ログ監視などが未実装

---

## 次に直すべき点（3つに絞る）
1. **サーバー側推論APIを導入する**
   - まずは FastAPI などで `/predict` を作り、CLIP系や物体検出モデルに接続する
2. **データ基盤を整える**
   - 正解ラベル付き画像データを整備し、ブランド・モデルごとの精度を測る
3. **信頼性/安全性を上げる**
   - 入力バリデーション、レート制限、監視、エラーレポートを導入する

## GitHub Pages / Vercel で公開しやすい構成について
このリポジトリは静的ファイルのみで構成しており、**そのまま GitHub Pages / Vercel に公開可能**です。

- `index.html` をエントリにした静的サイト
- `manifest.webmanifest` に `id/start_url/scope` を `./` で指定（サブパス公開に強い）
- `sw.js` を含むPWA構成
- GitHub Pages 向けに `.nojekyll` を配置
- Vercel 向けに `vercel.json` を配置（`sw.js` のキャッシュ制御）

## iPhone (Safari) でホーム画面に追加する手順
1. 公開URLを Safari で開く（HTTPS 必須）。
2. 画面下の共有ボタンをタップ。
3. **「ホーム画面に追加」**を選択。
4. 名前を確認して「追加」。
5. ホーム画面アイコンから起動し、カメラ許可後に利用開始。
